#+title: wayland-stt
#+description: Talk anywhere. Words appear at your cursor.

[[https://github.com/kyutai-labs/moshi][Kyutai STT]] powered push-to-talk for Wayland. Hold a key, speak, release — words stream into any text field in real time. ~500 ms latency. Runs entirely on your hardware.

#+begin_example
  hold key → speak → words appear live ✦ no cloud, no subscription, no latency
#+end_example

* Why

Every other speech-to-text tool either sends your audio to a server you don't control, or transcribes in batch after you're done talking. wayland-stt does neither. It streams words as you speak, locally, on your GPU.

- *Real-time* — words appear while you're still talking
- *Private* — audio never leaves your machine or network
- *Native Wayland* — works on niri, Hyprland, Sway, COSMIC, and any wlroots-based compositor
- *Set and forget* — systemd services start with your session

* Client / Server

wayland-stt separates the heavy GPU inference from the lightweight client. One beefy machine can serve your entire network.

#+begin_example
┌─────────────────────────────┐         ┌─────────────────────────────┐
│  desktop / laptop / phone   │         │      GPU workstation        │
│                             │   ws    │                             │
│  mic → wayland-stt ─────────┼────────►│  moshi-server (CUDA)        │
│       (records + types)     │◄────────┼─── word tokens              │
│                             │ tailscale│  model in VRAM (~2.4 GB)   │
└─────────────────────────────┘         └─────────────────────────────┘
       any Wayland device                   one NVIDIA GPU is enough
#+end_example

** Single machine (GPU + client together)

#+begin_src nix
services.wayland-stt = {
  enable = true;
  cudaCapability = "8.6";  # RTX 3060-3090
};
#+end_src

** Server (GPU machine, shared on your network)

#+begin_src nix
services.wayland-stt = {
  enable = true;
  cudaCapability = "8.6";
  serverAddr = "0.0.0.0";  # accept connections over Tailscale
};
#+end_src

** Client (any machine, no GPU needed)

#+begin_src nix
services.wayland-stt = {
  enable = true;
  enableServer = false;
  serverUrl = "ws://workstation:8098";  # point at your GPU machine
};
#+end_src

* Quick start

Add the flake input and import the Home Manager module:

#+begin_src nix
# flake.nix
{
  inputs.wayland-stt.url = "github:user/wayland-stt";

  # in your home-manager config:
  imports = [ inputs.wayland-stt.homeManagerModules.default ];

  services.wayland-stt = {
    enable = true;
    cudaCapability = "8.6";
  };
}
#+end_src

Rebuild, bind a key to toggle recording, done:

#+begin_src kdl
// niri
binds {
    Mod+V       { spawn "systemctl" "--user" "kill" "-s" "USR1" "wayland-stt.service"; }
    Mod+Shift+V { spawn "systemctl" "--user" "kill" "-s" "USR2" "wayland-stt.service"; }
}
#+end_src

The model (~2.4 GB) downloads from HuggingFace on first run.

* Requirements

** Server (runs inference)

- NVIDIA GPU with CUDA
- NixOS (flake-based)

** Client (records + types)

- Wayland compositor
- PipeWire
- NixOS (flake-based)
- No GPU needed when connecting to a remote server
