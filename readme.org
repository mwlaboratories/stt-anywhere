#+title: stt-anywhere
#+description: Speech-to-text anywhere — desktop, AR glasses, any device.

[[https://github.com/kyutai-labs/moshi][Kyutai STT]] powered speech-to-text service. Streams words in real time from any device on your network — Wayland desktops via push-to-talk, AR glasses via WebSocket relay. Runs entirely on your hardware.

#+begin_example
  speak → words appear live ✦ no cloud, no subscription, no latency
#+end_example

** Wayland push-to-talk

Press a key, speak, release. Words stream into whatever text field has focus via =wtype=. Works on niri, Hyprland, Sway, COSMIC, and any wlroots-based compositor.

** AR glasses relay

The relay server accepts WebSocket audio from remote clients (e.g. [[https://github.com/mwlaboratories/deltaclaw][deltaclaw]] on Even Hub glasses) and returns transcribed words as JSON. Also proxies Discord API calls so the glasses app can browse channels and send messages by voice.

* Why

- *Real-time* — words appear while you're still talking
- *Private* — audio never leaves your machine or network
- *Multi-device* — one GPU serves desktops, laptops, glasses, anything
- *Set and forget* — systemd services start with your session

* Quick start

Add the flake input and import the Home Manager module:

#+begin_src nix
# flake.nix
{
  inputs.stt-anywhere.url = "github:mwlaboratories/stt-anywhere";

  # in your home-manager config:
  imports = [ inputs.stt-anywhere.homeManagerModules.default ];

  services.stt-anywhere = {
    enable = true;
    cudaCapability = "8.6";  # RTX 3060-3090
  };
}
#+end_src

Rebuild, bind a key to toggle recording, done:

#+begin_src kdl
// niri
binds {
    Mod+Space   { spawn "systemctl" "--user" "kill" "-s" "USR1" "stt-anywhere.service"; }
    Mod+Shift+V { spawn "systemctl" "--user" "kill" "-s" "USR2" "stt-anywhere.service"; }
}
#+end_src

The model (~2.4 GB) downloads from HuggingFace on first run.

* Configuration examples

** Single machine (GPU + client together)

#+begin_src nix
services.stt-anywhere = {
  enable = true;
  cudaCapability = "8.6";
};
#+end_src

** GPU server (shared on your network)

#+begin_src nix
services.stt-anywhere = {
  enable = true;
  cudaCapability = "8.6";
  serverAddr = "0.0.0.0";       # accept moshi connections over Tailscale
  relayPort = 8099;              # WebSocket relay + Discord proxy for glasses
  discordGuildId = "123456789";  # Discord server ID
  discordTokenFile = "/run/secrets/discord-token";  # bot token
};
#+end_src

** Client (laptop, no GPU)

#+begin_src nix
services.stt-anywhere = {
  enable = true;
  enableServer = false;
  serverUrl = "ws://workstation:8098";
};
#+end_src

* Discord proxy

When =discordTokenFile= and =discordGuildId= are set, the relay server exposes a Discord API proxy:

| Endpoint                                  | Method | Description        |
|-------------------------------------------+--------+--------------------|
| =/api/discord/channels=                   | GET    | List text channels |
| =/api/discord/channels/:id/messages=      | GET    | Fetch messages     |
| =/api/discord/channels/:id/messages=      | POST   | Send a message     |

The bot token stays on your server — clients never see it.

** Setting up the Discord bot

1. [[https://discord.com/developers/applications][Discord Developer Portal]] — New Application — name it "deltaclaw"
2. Bot tab — Reset Token — save the token
3. OAuth2 — URL Generator — select =bot= scope + =Send Messages= + =Read Message History=
4. Open the generated URL to invite the bot to your server
5. Right-click your server name — Copy Server ID (enable Developer Mode in Discord settings first)

* Requirements

** Server (runs inference)

- NVIDIA GPU with CUDA
- NixOS (flake-based)

** Client (records + types)

- Wayland compositor + PipeWire
- NixOS (flake-based)
- No GPU needed when connecting to a remote server
